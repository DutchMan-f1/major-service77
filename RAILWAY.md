# Деплой на Railway (для теста)

Сайт упакован в Docker (PHP 8.3 + Apache + SQLite). Railway соберёт образ из
`Dockerfile` и запустит. База (SQLite) и фото товаров уже внутри — отдельно ничего
подключать не надо.

## Что уже готово в проекте
- `Dockerfile` — образ PHP+Apache, слушает порт Railway (`$PORT`), включён mod_rewrite и SQLite.
- `.dockerignore` — лишнее в сборку не попадает.
- `railway.json` — указывает Railway собирать из Dockerfile.
- `wp-config.php` — адрес сайта берётся от домена автоматически (Railway-домен подхватится сам, HTTPS тоже).

## Шаги

### 1. Установить Railway CLI
Нужен установленный Node.js. Затем в терминале:
```powershell
npm i -g @railway/cli
```
(Альтернатива без Node: `scoop install railway` или скачать бинарник со страницы Railway CLI.)

### 2. Войти
```powershell
railway login
```
Откроется браузер — залогинься (можно через GitHub).

### 3. Создать проект (из корня проекта)
```powershell
cd C:\Users\Arisi\Downloads\wordpress-7.0-ru_RU
railway init
```
Задай любое имя проекта.

### 4. Задеплоить
```powershell
railway up
```
CLI загрузит проект (~600 МБ — в основном фото товаров, первый раз может идти несколько минут)
и соберёт образ. Дождись строки, что деплой успешен.

### 5. Получить публичную ссылку
```powershell
railway domain
```
Сгенерируется адрес вида `https://имя-xxxx.up.railway.app`. Открой его — сайт живой.
(То же можно сделать в дашборде Railway: сервис → Settings → Networking → Generate Domain.)

## Важно про тест
- **Данные временные.** Заказы и правки в CMS живут, пока сервис работает. При повторном
  `railway up` (новый деплой) всё сбрасывается к исходному состоянию из образа. Для показа/теста
  это нормально. Нужно постоянное хранение — добавим Railway Volume (скажи, помогу).
- **Кредит Railway.** На бесплатном триале ограниченный кредит — для теста хватает, но следи за расходом.
- **ЮKassa** для теста не настраиваем (нужен реальный магазин + ключ). Оплата будет через плейсхолдер.

## Если что-то пошло не так
- Логи сборки/запуска: `railway logs` или в дашборде.
- Белый экран / ошибка 500 — почти всегда видно в логах; пришли их, разберёмся.

## Альтернатива через GitHub

Но из-за 515 МБ фото репозиторий получается тяжёлым — для быстрого теста
