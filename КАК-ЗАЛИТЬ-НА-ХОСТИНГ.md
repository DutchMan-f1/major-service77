# Как залить сайт MAJOR SERVICE77 на хостинг

Проект работает на **SQLite** — вся база лежит одним файлом внутри сайта
(`wp-content/database/.ht.sqlite`), отдельный MySQL не нужен. Поэтому перенос =
скопировать папку сайта на хостинг. Конфиг уже сделан «переносимым»: адрес сайта
подхватывается автоматически от домена, на котором он открыт (ничего в базе править не надо).

---

## 1. Что нужно от хостинга
- **PHP 8.1+** (у нас разрабатывалось на 8.3).
- Расширение **pdo_sqlite** (почти везде включено).
- **mod_rewrite** (Apache/LiteSpeed) — для «человекопонятных» ссылок. У большинства
  виртуальных хостингов (ISPmanager, cPanel, Timeweb, Beget, reg.ru и т.п.) это Apache/LiteSpeed.
- Права на запись в `wp-content/uploads` и `wp-content/database`.
- Желательно сразу **бесплатный SSL** (Let's Encrypt) — для HTTPS и оплаты ЮKassa.

> Если на хостинге вдруг запрещён SQLite (редко на дешёвых тарифах) — напиши,
> переведём базу на MySQL, это отдельный шаг.

## 2. Что заливать
Заливаем **содержимое папки `wordpress/`** (не саму папку) в корень сайта — обычно это
`public_html` / `www` / `htdocs`. То есть `index.php`, `wp-admin`, `wp-content`,
`wp-config.php`, `.htaccess` должны оказаться прямо в корне домена.

Способы:
- **Файловый менеджер хостинга**: заархивировать `wordpress/` в zip, загрузить, распаковать в корень.
- **FTP/SFTP** (FileZilla): перетащить содержимое `wordpress/` в корень.

Размер ~600 МБ (в основном фото товаров в `wp-content/uploads`). Через файловый менеджер
zip-ом быстрее, чем по FTP по одному файлу.

`router.php` в корне не нужен на хостинге (он только для локального запуска) — можно удалить, можно оставить, он не мешает.

## 3. База данных
Ничего импортировать **не надо**. База SQLite уже внутри (`wp-content/database/.ht.sqlite`)
и приедет вместе с файлами. Просто убедись, что папка `wp-content/database` загрузилась
полностью и доступна на запись (права 0755 на папку, 0644 на файл; если будут ошибки записи — поставь 0775/0664).

## 4. Ссылки / постоянные ссылки
В корень уже положен `.htaccess` со стандартными правилами WordPress — на Apache/LiteSpeed
ссылки заработают сразу.

- **nginx** (VPS): `.htaccess` не читается, добавь в конфиг сайта:
  ```nginx
  location / { try_files $uri $uri/ /index.php?$args; }
  location ~ /wp-content/database/ { deny all; }   # закрыть базу от интернета
  ```

## 5. HTTPS
Включи SSL-сертификат в панели хостинга (обычно кнопка «Выпустить Let's Encrypt»).
Больше ничего не нужно — сайт сам определит https (в `wp-config.php` это уже учтено, в т.ч. за прокси/CDN).

## 6. Синхронизация с baz-on (важно для каталога)
Автообновление товаров с baz-on раз в 30 минут работает через WP-Cron, но у нас
он отключён от «пинков» посетителями (`DISABLE_WP_CRON = true`), чтобы не тормозил сайт.
Поэтому на хостинге заведи **cron-задание** (раздел «Планировщик задач» / Cron):

```
*/15 * * * * wget -q -O /dev/null "https://ВАШ-ДОМЕН/wp-cron.php?doing_wp_cron"
```
(каждые 15 минут дёргает планировщик; сам импорт baz-on запускается по своему 30-минутному расписанию.)

## 7. Оплата ЮKassa
После переезда в **WooCommerce → Оплата → ЮKassa** ничего в коде менять не надо, но в
личном кабинете ЮKassa обнови **URL для HTTP-уведомлений** на новый домен:
`https://ВАШ-ДОМЕН/wc-api/mjr_yookassa/` (плагин показывает актуальный адрес в своих настройках).
Вебхуки требуют HTTPS — поэтому п.5 обязателен.

## 8. Безопасность (5 минут после заливки)
- Зайди в админку `https://ВАШ-ДОМЕН/wp-admin/` и **смени пароль администратора**
  (сейчас стоит рабочий/тестовый).
- Проверь, что база недоступна из браузера: открой
  `https://ВАШ-ДОМЕН/wp-content/database/.ht.sqlite` — должна быть ошибка 403/404
  (на Apache её закрывает и префикс `.ht`, и наш `.htaccess`).
- При желании можно сгенерировать новые «соли» ключей (строки AUTH_KEY… в `wp-config.php`) —
  это разлогинит всех, но повысит безопасность.

## 9. Проверка после переезда
1. Открывается главная `https://ВАШ-ДОМЕН/`.
2. Каталог `/shop/`, карточка товара, фильтры по авто.
3. Вход/регистрация, корзина, оформление.
4. Личный кабинет.
5. Админка → «Управление сайтом» (CMS) открывается.
